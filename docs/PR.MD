# PR 규칙

## PR 생성 후 CI 모니터링 (필수)

- **PR을 생성한 후, 모든 CI 체크가 통과할 때까지 작업은 완료된 것이 아니다.**
- PR 생성 직후 아래 모니터링 절차를 실행한다.

### 모니터링 절차

1. PR 생성 후 `gh pr checks <PR번호> --watch` 또는 주기적 `gh pr checks <PR번호>` 로 CI 상태를 확인한다.
2. 모든 체크가 통과하면 작업 완료로 간주한다.
3. **하나라도 실패하면:**
   - 실패한 체크의 로그를 확인한다 (`gh run view <run-id> --log-failed`).
   - 원인을 분석하고 수정한다.
   - 수정 커밋을 push한 뒤, 다시 모니터링을 시작한다.
   - 모든 체크가 통과할 때까지 이 과정을 반복한다.
4. CI가 전부 통과하기 전에 사용자에게 "작업 완료"를 보고하지 않는다.

### CI 모니터링 크론 (권장)

PR을 생성한 세션에서 아래 크론을 등록하면 자동으로 CI 상태를 추적할 수 있다.

```
CronCreate:
  cron: "*/2 * * * *"
  recurring: true
  prompt: |
    현재 PR의 CI 상태를 확인하라:
    1. `gh pr checks` 명령으로 모든 체크 상태를 조회한다.
    2. 모든 체크가 pass이면 사용자에게 "CI 전체 통과"를 알리고 크론을 삭제한다.
    3. 실패한 체크가 있으면 로그를 확인하고, 수정 가능한 문제면 즉시 수정 후 push한다.
    4. 아직 진행 중(pending/in_progress)이면 다음 주기까지 대기한다.
```

## PR 작성 규칙

- PR 제목은 70자 이내로 작성한다.
- PR 본문에는 Summary와 Test plan을 포함한다.
- 커밋 규칙은 [docs/COMMIT.MD](./COMMIT.MD)를 따른다.
