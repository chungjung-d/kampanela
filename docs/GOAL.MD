# 🔔 kampanela의 핵심 목적 (Core Purpose)

이 레포지토리는 개별 서비스의 코드를 담는 곳이 아니라, **모든 개발 프로세스를 자동화하고 시각화하는 '메타 시스템'**이다.

## 1. Multi-Repo Orchestration (지휘)

- **목적**: 여러 개의 독립된 레포지토리(PEP, Pazizik, Tuk 등)에 흩어져 있는 Claude Code 인스턴스들을 하나의 팀으로 묶는다.
- **역할**: "전체 시스템의 상태를 변경해"라는 상위 레벨의 명령을 받아, 각 레포지토리의 에이전트들에게 적절한 태스크를 분배하고 동기화한다.

## 2. Experimental Agent Teams UI (시각화)

- **목적**: 터미널 속의 텍스트로만 존재하는 에이전트들의 활동을 도트 그래픽 오피스로 시각화한다.
- **역할**: `~/.claude/teams/`의 상태 데이터를 실시간으로 읽어와 에이전트의 '생각-협업-실행' 과정을 인간이 직관적으로 관찰할 수 있는 '라이브 뷰'를 제공한다.

## 3. Shared Brain & Harness Hub (지식 관리)

- **목적**: 각 레포지토리의 하네스(`.claude.md`)와 subagent 정의를 중앙에서 관리한다.
- **역할**: 특정 레포지토리에 종속되지 않는 공통 컨벤션, 아키텍처 가이드라인, MCP 서버 연결 설정을 관리하며, 새로운 에이전트 인턴이 투입될 때 즉시 업무에 적응할 수 있는 '온보딩 패키지'를 제공한다.

## 작업 범위 준수 규칙

- 본 레포지토리는 위 3가지 축(오케스트레이션 / 시각화 / 공유 브레인) 외의 기능을 직접 구현하지 않는다.
- 특정 서비스의 비즈니스 로직은 해당 서비스 레포에서 다루고, 여기서는 그들을 연결·관찰·지원하는 메타 레이어만 다룬다.

## 최종 원칙

- **모든 변경은 위 세 축 중 어느 것을 강화하는지 명시할 수 있어야 한다.**
- 축과 연결되지 않는 변경은 계획 단계에서 범위를 재검토한다.
